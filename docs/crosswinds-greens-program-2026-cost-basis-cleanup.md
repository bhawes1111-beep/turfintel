# Crosswinds Greens Program 2026 — Cost Basis Cleanup Workflow

Review-first workflow for filling in the missing cost basis on the
Crosswinds Greens Program 2026 products. **Nothing here is applied
automatically.** This phase prepares a *suggested* draft for Bryan to
review, then apply by hand through the existing UI.

## What this is built on

- [Product audit](crosswinds-greens-program-2026-product-audit.md) —
  Phase 7T.1: 66 unique products, 46 exact inventory matches, 66 missing
  cost basis (migration 0021 set every `cost_per_unit` to NULL).
- [Vendor pricing reference](crosswinds-greens-program-2026.md) — the
  "Vendor spend + rebate reference" section. This is the **only** price
  source. It is not stored in D1.
- Draft data:
  [`crosswinds-greens-program-2026-cost-basis-draft.json`](crosswinds-greens-program-2026-cost-basis-draft.json),
  generated by
  [`scripts/prepare-crosswinds-cost-basis-draft.mjs`](../scripts/prepare-crosswinds-cost-basis-draft.mjs)
  and validated by
  [`scripts/check-crosswinds-cost-basis-draft.mjs`](../scripts/check-crosswinds-cost-basis-draft.mjs).

> **Inventory stock is not deducted from planned spray programs.**
> Cost basis is a per-inventory-row reference price only; setting it
> does not consume stock, create inventory usage, or touch any spray
> record. Deduction happens solely through completed Spray Records.

## How to review the draft

1. Regenerate the draft (read-only preview):
   ```
   node scripts/prepare-crosswinds-cost-basis-draft.mjs
   ```
   Add `--write` to refresh the JSON file, `--json` for raw output.
2. Validate it:
   ```
   node scripts/check-crosswinds-cost-basis-draft.mjs
   ```
3. Open the JSON and read each entry's `confidence` + `notes`:
   - **exact** — clean purchase unit (gal / lb) AND the program name
     matches an inventory row exactly. Safe to apply as-is after a
     sanity check.
   - **alias-review** — a suggestion exists but the inventory row's name
     differs from the program name. Confirm the names refer to the same
     product before linking.
   - **manual-review** — no clean per-unit price could be derived (priced
     by case / bag / bottle, or bundled). `suggestedCostPerUnit` is
     `null`. Bryan must work out the per-unit cost from the package size.
4. Apply approved values **by hand** in the existing UI:
   Inventory → Products → (product) → cost basis field. That goes
   through the Phase 7J.1 narrow PATCH endpoint (`cost_source = 'manual'`,
   audit-logged via Phase 7M.1). No bulk apply script exists yet — that
   is intentional for this phase.

## Products needing alias review

The current draft has **0** alias-review entries: every product with a
clean per-unit price also matched an inventory row by exact name. Name
mismatches that DO exist (e.g. `Kickdrum 0-0-29 K Acetate` vs inventory

## Products needing manual unit conversion (35)

These are priced by **case / bag / bottle / pack**, or bundled into a
vendor solution with no standalone price. `suggestedCostPerUnit` is
`null` — the program rate units are gal/acre, lb/acre, oz/1000, etc., and
the document does not state the package size (e.g. gallons per case,
pounds per bag), so a per-unit price cannot be derived without guessing.

Examples:
- **Granular (by bag):** VerdeCal Lime/Gypsum, KMag, Ecolite,
  MycoReplenish, 5-4-5 Greens Grade, Vereens 13-2-13, Potassium Nitrate,
  Calcium Nitrate, Epsom Salt — per-bag price is known, but bag weight is
  not stated, so per-lb cannot be computed.
- **Liquid (by case):** Harmony, Ampliphy 18, Microtone, PowerChord,
  BioRhythym, Double Bass Kelp, Prize Phiter, KickDrum, Rootnote, Sea
  Sugar, Sweet Heat, Hydra 30 Plus, Excalibur, Oars PS — gallons per case
  not stated.
- **By bottle / bundled:** Fosetyl Al (27 bottles), Segway (by bottle),
  Daconil Action / Secure Action / Appear / Appear II / Ascernity
  (bundled in the Syngenta Greens Foundation Solution — no standalone $).
- **Prothioconazole** has a clean $1320/gal suggestion but its inventory
  row is named "Prothioconazole (generic Densicor)", so it is
  manual-review until the name is reconciled (NOT auto-merged).

For each: divide the package price by the real package size (from the
invoice or label) to get the per-gal / per-lb cost, then enter it
manually.

## DO NOT MERGE

`Ampliphy 18` and `Veriphy 18` are **separate products** and appear as
separate draft entries. Never merge them or link one's cost basis to the
other. The check script enforces this.

## Recommended cleanup order

1. **Apply the 13 `exact` entries** — fastest win, lowest risk. Confirm
   each per-unit number against the invoice, then enter via the UI.
2. **Reconcile name mismatches** so more products can become `exact` on
   the next draft regeneration (Kickdrum/KickDrum, Fame/Fame SC,
   Prothioconazole/Densicor, Harmony/Root Harmony).
3. **Work out by-bag granular per-lb costs** from bag weights — these are
   high-volume program inputs, so the cost basis matters most here.
4. **Work out by-case liquid per-gal costs** from case sizes.
5. **Resolve bundled Syngenta line items** (Daconil Action, Secure
   Action, Appear/Appear II, Ascernity) — the document bundles them, so
   get standalone pricing from the vendor.
6. **Leave the DO-NOT-MERGE pair separate**; price each independently.

After each manual entry, the Spray Program Cost report's "Missing cost
basis" count drops and "Estimated items" rises automatically — no rebuild
needed.
