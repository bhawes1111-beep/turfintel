3. Warnings ⚠️

Partial Transaction Safety:

  • BuildSpraySheet calls recordInventoryUsage() once per product via Promise.allSettled()
  • Each call is atomic, but no outer transaction wraps all deductions
  • Risk: If product #3 of 5 fails, products #1 and #2 have already committed
  • Mitigation: allSettled() allows successes to commit; failures show toast warnings
  • Acceptable for current feature set (spray sheet has "skipped" mechanism for unit mismatches)

No Usage Dedup:

  • inventory_usage has source_id column but no UNIQUE constraint
  • Risk: Double-deduction if spray record editing lands without dedup logic
  • Safe for now: BuildSpraySheet creates new spray_record on every commit (no edit flow exists)
